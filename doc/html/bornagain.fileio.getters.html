bornagain.fileio.getters module¶

class bornagain.fileio.getters.FrameGetter[source]¶

Bases: object

Experimental - a generic interface for serving up data frames. This should basically just serve up dictionaries that have entries for diffraction data, peak positions, x-ray source, and so on.

This class is just a template - you’re not supposed to use it. You may want to subclass it so that the assumed methods are available, even if they just return None by default.

Minimally, a “frame getter” should have a simple means to provide infomation on how many frames there are, and methods for getting next frame, previous frame, and arbitrary frames.

Once could imagine making things fast by having parallel threads or processes that are pulling data of disk and cleaning it up prior to serving it up to a top-level program. Or, the getter just serves up raw data.

We could also have getters that serve up simulated data.

It is expected that these getters need a lot of customization, since we simply cannot avoid the many ways in which data is created and stored…
